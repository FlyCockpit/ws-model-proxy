# Homebrew tap

This folder documents how the `wsmp` Homebrew formula is published. The actual
tap is the separate GitHub repository:

```text
https://github.com/FlyCockpit/homebrew-tap
```

Users install from that tap with:

```sh
brew install flycockpit/tap/wsmp
```

## One-time setup

1. Keep `FlyCockpit/homebrew-tap` public so Homebrew can clone it without auth.
2. In `FlyCockpit/ws-model-proxy`, create a protected `release` environment
   and add an environment secret named `HOMEBREW_TAP_TOKEN`. Use a
   fine-grained GitHub token with `contents:write` access to
   `FlyCockpit/homebrew-tap`. The default `GITHUB_TOKEN` cannot push to a
   different repository.

## Release behavior

The root `Release` workflow in this repository does the full publish path.
The Homebrew publish job uses the protected `release` environment:

1. Builds and pushes the app container to GHCR.
2. Builds cross-platform `wsmp` artifacts with `dist`.
3. Creates the GitHub Release and uploads installers, checksums, archives, and
   the generated `wsmp.rb` formula.
4. Checks out `FlyCockpit/homebrew-tap`, copies `wsmp.rb` to
   `Formula/wsmp.rb`, commits it, and pushes to `main`.

The formula is generated by cargo-dist from `dist-workspace.toml`; do not edit
it by hand. If the tap name changes, update `tap = "FlyCockpit/homebrew-tap"`
in `dist-workspace.toml` and review the root release workflow.

## How users install

```sh
brew install flycockpit/tap/wsmp
brew upgrade wsmp
```

Private repositories do not work with the one-line `brew install` flow unless
the user has GitHub/Homebrew auth configured. Keep both the source repo and the
tap public for the smooth install path.
